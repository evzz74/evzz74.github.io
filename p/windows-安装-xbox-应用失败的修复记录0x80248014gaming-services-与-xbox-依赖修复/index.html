Windows 安装 Xbox 应用失败的修复记录：0x80248014、Gaming Services 与 Xbox 依赖修复

一、问题背景

在 Windows 上安装 Xbox 应用时，遇到以下问题：

  • Microsoft Store 安装 Xbox 应用失败；
  • 绕过 Store 下载 Xbox 安装包后仍然安装失败；
  • winget 安装 Microsoft Store 程序包时报错；
  • 安装或更新 Microsoft Store 程序包时报错：
1

0x80248014

同时还出现过 winget 访问 msstore 源失败的问题：

1

0x8a15005e : 服务器证书与任何预期值都不匹配

后续经过排查发现，核心问题并不是 Xbox 安装包本身，而是以下几个组件存在异常：

  1. Gaming Services 服务注册异常；
  2. Xbox 新版应用 Microsoft.GamingApp 已经暂存，但没有注册到当前用户；
  3. 旧版 Microsoft.XboxApp 存在残留；
  4. Xbox 相关依赖组件需要重新注册或启用。

二、确认 winget 与 App Installer 状态

首先检查 winget 和 App Installer 是否正常：

1
2
3

winget --version
winget source list
Get-AppxPackage Microsoft.DesktopAppInstaller -AllUsers

本次排查中的输出显示：

1
2
3

winget version: v1.11.430
Microsoft.DesktopAppInstaller: 1.26.430.0
Status: Ok

这说明：

  • winget 本身正常；
  • App Installer 没有明显损坏；
  • 问题不在 winget 主程序，而更可能在 Microsoft Store / Windows Update / Xbox 相关 AppX 包注册链路。

三、处理 msstore 源证书校验问题

安装过程中出现过：

1

0x8a15005e : 服务器证书与任何预期值都不匹配

该问题会导致 winget 无法正常访问 msstore 源。

本次使用的处理方式是临时启用 Microsoft Store 证书固定绕过：

1

winget settings --enable BypassCertificatePinningForMicrosoftStore

之后再执行：

1

winget source update

这个步骤的作用是绕过 msstore 源的证书固定校验异常，使后续 Microsoft Store 包安装能够继续进行。

注意：该选项建议只作为临时修复手段。问题解决后可以关闭：

1

winget settings --disable BypassCertificatePinningForMicrosoftStore

四、修复 Gaming Services

Xbox 应用高度依赖 Gaming Services。

本次排查中，最初使用以下命令检查服务时发现服务不存在：

1

Get-Service GamingServices,GamingServicesNet

报错类似：

1
2

找不到任何服务名称为“GamingServices”的服务。
找不到任何服务名称为“GamingServicesNet”的服务。

这说明 Gaming Services 没有正确注册为系统服务。

1. 卸载残留 Gaming Services

使用管理员 PowerShell 执行：

1

Get-AppxPackage Microsoft.GamingServices -AllUsers | Remove-AppxPackage -AllUsers

然后清理残留服务注册表项：

1
2

Remove-Item -Path "HKLM:\System\CurrentControlSet\Services\GamingServices" -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item -Path "HKLM:\System\CurrentControlSet\Services\GamingServicesNet" -Recurse -Force -ErrorAction SilentlyContinue

执行完成后重启电脑。

2. 重新安装 Gaming Services

重启后，重新安装 Gaming Services。

可以使用：

1

winget install --id 9MWPM2CQNLHN --source msstore

如果 Microsoft Store 链路仍然异常，也可以通过安装包方式安装 Gaming Services。

本次最终确认有效的结果是：

1

Get-Service GamingServices,GamingServicesNet

能够正常显示服务，说明 Gaming Services 已经恢复。

五、检查 Xbox 应用主程序状态

新版 Xbox 应用对应的 AppX 包不是旧的 Microsoft.XboxApp，而是：

1

Microsoft.GamingApp

检查命令：

1

Get-AppxPackage Microsoft.GamingApp -AllUsers

本次排查中，发现 Microsoft.GamingApp 已经存在：

1
2
3
4

Name: Microsoft.GamingApp
Version: 2508.1001.27.0
Status: Ok
PackageUserInformation: S-1-5-18: Staged

这里的关键点是：

1

Staged

这表示 Xbox 应用包已经下载或暂存到了系统中，但还没有注册到当前用户账户。

因此，虽然包存在，但当前用户仍然无法正常使用 Xbox 应用。

六、将 Microsoft.GamingApp 注册到当前用户

这是本次修复中非常关键、并且最终有效的一步。

管理员 PowerShell 执行：

1
2
3

Get-AppxPackage Microsoft.GamingApp -AllUsers | Foreach {
    Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppxManifest.xml"
}

执行后再次检查：

1

Get-AppxPackage Microsoft.GamingApp

如果当前用户下能查询到 Microsoft.GamingApp，说明 Xbox 应用已经注册到当前用户。

随后在开始菜单搜索：

1

Xbox

此时 Xbox 应用已经可以正常打开。

七、处理旧版 Microsoft.XboxApp 残留

排查过程中还发现旧版 Xbox 应用包存在：

1

Get-AppxPackage Microsoft.XboxApp -AllUsers | Select Name, PackageFullName, InstallLocation, Status

输出中包含：

1

Microsoft.XboxApp_48.104.4001.0_x64__8wekyb3d8bbwe

尝试卸载时出现：

1

错误为 0x80070002

说明该旧包存在残留或文件状态异常。

不过，新版 Xbox 应用的主程序是：

1

Microsoft.GamingApp

因此旧版 Microsoft.XboxApp 不是必须解决的核心问题。

本次验证有效的处理方式是移除旧版 XboxApp 的预装模板：

1

Get-AppxProvisionedPackage -Online | Where-Object DisplayName -eq "Microsoft.XboxApp" | Remove-AppxProvisionedPackage -Online

执行结果：

1
2

Online        : True
RestartNeeded : False

这表示旧版 XboxApp 的系统预装登记已被移除。

需要注意的是，不建议继续使用下面这种大范围卸载命令：

1

Get-AppxPackage *Xbox* | Remove-AppxPackage

因为它会匹配到 Windows 系统自带组件，例如：

1

Microsoft.XboxGameCallableUI

该组件是 Windows 的一部分，无法正常按用户卸载，强行卸载会报错。

八、重新注册 Xbox 依赖组件

Xbox 应用成功打开后，仍提示缺少依赖，例如：

1
2
3

Game Bar
Xbox Live
Xbox Identity Provider

这时需要重新注册现有 Xbox 依赖组件。

管理员 PowerShell 执行：

Microsoft.Xbox.TCUI               Xbox 登录/交互相关组件

九、检查 Xbox Live 相关服务

Xbox Live 不是单独的普通应用，而是依赖多个系统服务。

使用以下命令检查：

1

Get-Service XblAuthManager,XblGameSave,XboxGipSvc,XboxNetApiSvc | Select Name,Status,StartType

如果服务被禁用，可以设置为手动启动：

1
2
3
4

Set-Service XblAuthManager -StartupType Manual
Set-Service XblGameSave -StartupType Manual
Set-Service XboxGipSvc -StartupType Manual
Set-Service XboxNetApiSvc -StartupType Manual

然后尝试启动：

1
2
3
4

Start-Service XblAuthManager
Start-Service XblGameSave
Start-Service XboxGipSvc
Start-Service XboxNetApiSvc

十一、经验总结

这次问题的关键不在 Xbox 安装包本身，而是 Windows 的 AppX / Store / Xbox 依赖链路异常。

最终有效的关键点是：

  1. Gaming Services 必须真正注册成系统服务；
  2. Microsoft.GamingApp 不能只停留在 Staged 状态，需要注册到当前用户；
  3. 旧版 Microsoft.XboxApp 残留不一定需要强行删除；
  4. 不要盲目执行 Get-AppxPackage *Xbox* | Remove-AppxPackage，因为会误匹配系统组件；
  5. Xbox 应用打开后仍可能提示缺少 Game Bar、Xbox Live、Xbox Identity Provider，需要继续注册依赖组件和检查服务。

最终，通过重新安装 Gaming Services、注册 Microsoft.GamingApp、移除旧版 XboxApp 预装模板、重新注册 Xbox 依赖组件后，Xbox 应用成功打开。
