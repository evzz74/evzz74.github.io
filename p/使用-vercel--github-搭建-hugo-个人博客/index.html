使用 Vercel + GitHub 搭建 Hugo 个人博客

一、前言

很多人会选择使用 GitHub Pages 搭建个人博客，但在部分网络环境下访问速度可能不够理想。Vercel 是一个静态站点和前端项目部署平台，可以与 GitHub 仓库联动：当仓库内容更新后，Vercel 会自动重新部署网站。

本文整理了一套从零开始搭建个人博客的流程，主要使用以下工具：

  • GitHub：用于托管博客源码；
  • Vercel：用于部署静态网站；
  • Hugo：用于生成静态博客；
  • GitHub Desktop：用于更方便地同步本地修改。

说明：整理自博客园文章《使用Vercel+Github搭建个人博客》，原文写于 2020 年，Vercel 后台界面可能已经更新，但整体思路仍然可以参考。

二、准备工作

在开始之前，需要准备：

  1. 一个 GitHub 账号；
  2. 一个 Vercel 账号；
  3. 可选：一个自定义域名；
  4. 可选：安装 GitHub Desktop，方便后续提交和推送文件。

三、注册 GitHub 账号

如果还没有 GitHub 账号，需要先前往 GitHub 官网注册。注册完成后，后续 Vercel 会通过 GitHub 授权创建或导入仓库。

四、注册 Vercel 账号

  1. 打开 Vercel 注册页面：

    1

    https://vercel.com/signup

  2. 选择 Continue with GitHub。

  3. 页面跳转到 GitHub 授权界面后，点击 Authorize Vercel，允许 Vercel 访问 GitHub。

完成授权后，Vercel 账号就会与 GitHub 账号绑定。

五、创建 Hugo 博客项目

1. 导入项目

进入 Vercel 控制台后，点击 Import Project。

2. 选择模板

在模板选择页面中找到 Hugo 模板。Hugo 是一个静态网站生成器，适合用来搭建个人博客。

选择 Hugo 后，点击 Continue。

3. 安装 Vercel GitHub 应用

如果是第一次使用 Vercel 连接 GitHub，页面会提示安装 Vercel 的 GitHub 应用。

按照提示点击：

  1. Install Vercel for GitHub；
  2. 在 GitHub 授权页面点击 Install；
  3. 保持默认配置；
  4. 返回 Vercel 后继续下一步。

4. 创建 GitHub 仓库

在 Create Git Repository 页面：

  • 选择 GitHub；
  • 填写仓库名称；
  • 根据需要选择公开仓库或私有仓库；
  • 点击 Continue。

Vercel 会自动在 GitHub 中创建一个仓库，用来存放 Hugo 博客源码。

5. 部署项目

在 Import Project 页面保持默认配置，点击 Deploy。

部署完成后，点击 Visit 即可访问刚刚创建好的网站。

此时网站已经可以访问，但默认样式比较基础，后续可以通过配置主题进行美化。

六、配置和修改博客

1. 克隆仓库到本地

可以使用 GitHub Desktop 来管理博客项目：

  1. 打开 GitHub Desktop；
  2. 登录 GitHub 账号；
  3. 点击 File；
  4. 选择 Clone repository；
  5. 找到刚才创建的 Hugo 仓库；
  6. 点击 Clone。

克隆完成后，项目文件通常会保存在类似下面的位置：

1

C:\Users\你的用户名\Documents\GitHub\仓库名

2. 配置 Hugo 主题

进入项目目录后，可以按照所选 Hugo 主题的文档进行配置。

原文中推荐的主题是 Even：

1

https://github.com/olOwOlo/hugo-theme-even

配置主题时通常需要修改以下内容：

  • Hugo 配置文件；
  • 主题目录；
  • 文章目录；
  • 网站标题、作者、菜单、评论系统等信息。

具体配置方式需要以主题官方文档为准。

3. 提交修改

修改完成后，回到 GitHub Desktop：

  1. 在左下角 Summary 中填写本次修改说明，例如：

    1

    update theme

  2. 点击 Commit to master 或 Commit to main；

  3. 点击右上角 Push origin，将修改推送到 GitHub。

七、自动部署机制

当 GitHub 仓库发生变化后，Vercel 会自动检测更新并重新构建网站。

因此，后续更新博客文章时，可以直接将 Markdown 文件上传或提交到 Hugo 的文章目录，例如：

1

./content/posts

提交并推送到 GitHub 后，等待 Vercel 自动部署完成，网站内容就会更新。

八、绑定自定义域名

如果有自己的域名，可以在 Vercel 中绑定。

1. 添加域名

进入 Vercel 控制台，打开对应项目：

1

Project > Settings > Domains

在输入框中填写想要绑定的域名，例如：

1

test.example.com

如果域名还没有配置解析，Vercel 会提示需要添加 DNS 记录。

2. 配置 DNS 解析

根据 Vercel 的提示，在域名服务商处添加 CNAME 解析记录。

一般情况下，需要设置：

1
2
3

类型：CNAME
主机记录：你的子域名前缀
记录值：Vercel 提供的目标地址

不同域名服务商的后台界面略有区别，但核心操作都是添加对应的 DNS 解析记录。

3. 等待生效

DNS 配置完成后，回到 Vercel 控制台检查域名状态。

解析生效后，Vercel 会自动为域名申请 HTTPS 证书，通常使用 Let’s Encrypt 证书。

完成后，就可以通过自己的域名访问博客了。

十、注意事项

  1. 原文基于 2020 年的 Vercel 页面，当前按钮名称和页面位置可能有所变化。
  2. Hugo 主题配置要以主题官方文档为准，不同主题的目录结构和配置项可能不同。
  3. GitHub 默认分支可能是 main，而早期教程中常见的是 master。
  4. 如果绑定域名后不能立即访问，通常是 DNS 解析尚未完全生效。
  5. 后续写文章时，建议直接使用 Markdown 文件，并放入 Hugo 的文章目录。

十一、适合新手的建议

如果你只是想快速搭建一个可访问的个人博客，可以先完成以下最低配置：

  • 使用 Vercel 的 Hugo 模板创建项目；
  • 不急着绑定域名，先使用 Vercel 默认域名；
  • 先学会在 content/posts 中新增文章；
  • 熟悉自动部署流程后，再配置主题和自定义域名。

这样可以先保证博客成功上线，再逐步美化和完善。
