如何将本地 Git 默认分支从 master 改为 main

Apr 11, 2026
阅读时长: 1 分钟
loading... 次

场景一：把全局默认分支改为 main（适用于新项目）

通过以下设置，以后使用 git init 初始化全新项目时，默认就会直接使用 main 作为主分支。这里提供两种修改方式，效果完全一致。

方法 1：使用 Git 命令（推荐，快捷方便）

最简单的方法，只需在终端或命令行中执行以下命令即可：

1

git config --global init.defaultBranch main

方法 2：手动修改配置文件

如果更喜欢底层操作，可以直接修改 Git 的全局配置文件：

在 Windows 系统中，Git 的全局配置文件通常位于

1

C:\Users\<用户名>\.gitconfig。

使用文本编辑器打开该文件。

在配置文件中添加（或修改）以下内容：

1
2

[init]
    defaultBranch = main

经过上述修改，未来创建的所有新项目都将默认拥有 main 分支。

场景二：修改已创建项目的主分支为 main（适用于老项目）

以上方法只对未来的新项目生效，对于已经创建好的老项目则无能为力。因此，对于已存在的项目，我们需要逐个进行手动修改。

操作步骤如下：

  1. 切换到主分支 master

首先确保当前是否处于 master 分支上：

1

git checkout master
  1. 重命名分支为 main

使用带有 -M 参数的 branch 命令，将当前分支重命名：

1
2

Bash
git branch -M main

(注：其中 -M 的意思是强制移动或者重命名当前分支。)
