Claude Code CLI 完整安装与进阶指南

一、 为什么选择 Claude Code？

与常见的 VS Code 插件（如 Cursor 或 Copilot）不同，Claude Code 是一个运行在终端（Terminal）中的 Agentic CLI。

它不仅能“写代码”，更能“操作项目”：

  • 自主性：它能跨文件搜索、阅读代码库、执行测试、修复报错并直接提交 Git。
  • 环境感官：它能感知你的终端环境，执行编译命令（如 cargo build 或 npm run dev）。
  • 组合性：作为 Unix 风格的工具，它可以与其他命令行工具管道连接，完美融入自动化工作流。

二、 准备工作

在开始之前，请确保你的开发环境满足以下条件：

  1. 账号权限：需要 Claude Pro/Team/Enterprise 订阅，或 Anthropic Console API 余额（推荐开发者使用 API 模式，按量计费更灵活）。
  2. 操作系统：macOS 13+、Windows 10+ (PowerShell/WSL)、Linux (Ubuntu/Debian/Alpine)。
  3. 硬件：建议 4GB 以上内存；Windows 环境建议安装 Git for Windows。

三、 全平台安装详解

请根据你的操作系统选择最稳定的安装方式：

1. macOS / Linux / WSL (推荐脚本)

官方提供了一键安装脚本，能自动处理路径配置：

1

curl -fsSL https://claude.ai/install.sh | bash

如果你使用 Homebrew (macOS):

1

brew install --cask claude-code

2. Windows 原生环境

PowerShell (管理员模式):

1

irm https://claude.ai/install.ps1 | iex

WinGet:

1

winget install Anthropic.ClaudeCode

3. Node.js/npm 备选方案 (适用于各类环境)

在开始之前，您的系统需要安装好 Node.js。

  • 下载与安装：请前往 Node.js 官方网站 下载并安装最新版本的 Node.js（建议使用 v18 或更高版本）。根据您的操作系统下载对应的安装包。
  • 验证安装：安装完成后，打开您的终端（Terminal）或命令提示符（CMD），运行以下命令来验证。
1

node -v

如果你已经配置好了 Node.js 环境，可以使用 npm 全局安装：

1

npm install -g @anthropic-ai/claude-code

注：在中国大陆环境下，建议追加镜像源：--registry=https://registry.npmmirror.com。

1

npm install -g @anthropic-ai/claude-code  —-registry=https://registry.npmmirror.com

四、 初始化与身份验证

安装完成后，在你的项目根目录启动cmd终端，在终端界面输入：

1

claude
  1. OAuth 登录：终端会显示一个验证码并自动打开浏览器。登录你的 Claude 账号并授权。
  2. 工作区配置：如果你是 API 用户，Claude 会引导你创建一个专用的工作区以监控 Token 消耗。
  3. 项目扫描：它会自动读取 .gitignore，确保不会意外处理无关的二进制文件或依赖包。

五、 进阶：斜杠命令与 CLI 参数

1. 交互模式内的常用命令

  • /help：查看所有指令。
  • /clear：当上下文太长或想切换任务时，清空当前对话记录以节省 Token。
  • /compact：压缩长对话，保留核心上下文。
  • /config：配置默认模型或更新通道。
  • /status: 查看当前会话的工作区状态，包括已读取的文件上下文、Token 消耗量以及预估的 API 费用，方便你随时把控开发成本。
  • /model：无缝切换底层大模型。你可以根据任务难度，在能力最强的Opus模型和速度极快、成本极低的Haiku模型或第三方模型之间灵活切换，以平衡性能与费用。

2. 强大的 CLI 启动参数

你可以直接在 shell 中调用 Claude，无需进入交互界面：

  • 执行任务并退出：claude "fix the bug in main.py"
  • 多目录支持：claude --add-dir ../another-repo (处理跨仓库依赖时非常有用)。
  • 纯净模式：claude --bare -p "generate a README.md" (不加载历史记录，适合脚本调用)。

六、 维护与更新

  • 自动更新：脚本安装版会在启动时自动检测更新。
  • 手动强制更新：执行 claude update。
  • 查看成本：访问 Anthropic Console （第三方平台可通过官网或CC-Switch）可以查看具体的 Token 消耗和账单。

七、 避坑指南 (FAQ)

  1. 网络连接失败：如果遇到连接超时，请确保你的终端环境正确配置了 HTTP_PROXY 和 HTTPS_PROXY 环境变量。
  2. 权限报错：在 Linux/macOS 上，如果提示权限不足，请避免使用 sudo claude，而是通过 chown 修改安装目录的归属。
  3. 大文件处理：如果项目文件过多，建议通过 .claudeignore 排除非代码目录（如 node_modules、target 等）。
  4. 第三方API配置: 推荐CC-Switch一键配置，目前支持Claude Code、Codex、Gemini、OpenClaw等应用的多平台第三方API一键配置，仅需填入API-Key即可一键使用。
